1. HTML 파싱
=> 브라우저는 HTML 파일을 위에서 아래로 파싱합니다.
=> 각 요소는 DOM(Document Object Model) 트리로 변환됩니다.


2. CSS 파싱
=> HTML 파싱이 진행되는 동안 브라우저는 <link>, <style> 태그를 통해 CSS를 로드합니다.
=> CSS도 파싱되어 CSSOM 트리가 생성됩니다.
=> 해당 트리는 각 스타일 규칙을 나타내는 객체입니다.


3. DOM과 CSSOM 결합
=> DOM과 CSSOM은 결합되어 렌더 트리가 만들어집니다. (렌더 트리는 실제 페이지를 그리는 정보만 포함)
=> 이 떄, CSS의 스타일 규칙들이 DOM 요소에 적용됩니다.


4. 레이아웃 계산
=> 렌더 트리가 만들어지면, 브라우저는 각 요소의 정확한 위치와 크기를 계산합니다. 
=> 브라우저는 요소들이 화면 상에서 어디에 배치될지를 결정합니다.


5. 페인팅
=> 레이아웃이 계산되면, 브라우저는 각 요소를 화면에 그리기 시작합니다. 
=> 요소의 색상, 배경, 텍스트, 이미지 등이 실제로 화면에 나타납니다.


6. 컴포지팅

6-1. 레어어
=> 화면에 그려지는 각 요소나 그룹을 의미합니다.
=> 배경, 텍스트, 이미지, 버튼 등
=> 각 레이어는 독립적으로 그려지고, 개별적으로 애니메이션을 적용하거나, 변화할 수 있습니다.
