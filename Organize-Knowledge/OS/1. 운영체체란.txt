1. 운영체제
=> 사용자와 컴퓨터 하드웨어를 연결해주는 관리자 역할을 하는 소프트웨어입니다.
=> 사용자가 운영체제에게 명령을 내리고 운영체제는 하드웨어에게 작업을 분배하고 관리합니다.
=> 대표적으로 Windows, macOS, Linux, Android, IOS 등이 존재합니다.


1-1. 운영체제 주요 기능
1. 프로세스 관리 (Process Management)
2. 메모리 관리
3. 파일 시스템 관리 
4. 디스크 및 입출력 장치 관리
5. 사용자 인터페이스


1-2. 운영체제 구성 요소
1. 커널
2. 쉘
3. 파일 시스템


1-3. 운영체제의 종류
1. 사용자 수에 따른 분류
=> 단일 사용자 OS : 한 명의 사용자만 사용 가능 (MS-DOS)
=> 다중 사용자 OS : 여러 사용자가 동시에 사용 가능 (UNIX, Linux)

2. 태스크 처리 방식에 따른 분류
=> 단일 태스킹 OS: 한 번에 하나의 작업만 실행 (예: MS-DOS)
=> 멀티태스킹 OS: 여러 개의 프로세스를 동시에 실행 (예: Windows, Linux)

3. 목적에 따른 분류
=> 범용 OS: 일반 사용자용 (예: Windows, macOS, Ubuntu)
=> 실시간 OS (RTOS): 실시간 데이터 처리를 위한 OS (예: VxWorks, FreeRTOS)
=> 임베디드 OS: 소형 기기에서 사용되는 OS (예: Android, iOS, RTOS)


1-4. 운영체제의 핵심 개념
1. 멀티태스킹 : 한 번에 여러 프로세스를 실행하는 기능
2. 멀티프로세싱 : 여러 개의 CPU(코어)를 활용하여 여러 프로세스를 실행하는 방식
3. 멀티스레딩 : 하나의 프로세스 내부에서 여러 개의 스레드를 실행하는 방식
4. 동기화 : 여러 프로세스 또는 스레드가 공유 자원을 안전하게 사용할 수 있도록 하는 기술
5. 데드락 : 두 개 이상의 프로세스가 서로의 자원을 기다리면서 무한정 대기하는 상태
