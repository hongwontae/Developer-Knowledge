1. OSI 계층 데이터 전달

1-1. 송신 측
=> 7 layer ~ 1 layer 각 계층을 내려가면서 헤더(제어 정보)가 추가됩니다.
(5~6 계층은 송신할 떄는 생략되거나 애플이케이션에 통합되는 경우가 많습니다.)
=> HTTP 메시지 -> TCP 세그먼트 -> IP 패킷 -> 이더넷 프레임 -> 비트
([이더넷 헤더][IP 헤더][TCP 헤더][HTTP 데이터] → 비트 스트림)
=> 최종적으로 물리적 신호(1계층)로 변환되어 네트워크를 통해 전송합니다.

1-2. 수신 측
=> 송신 측의 데이터가 수신 측에 도달하면 1계층부터 헤더를 분석하고 원본 데이터를 복원합니다.
=> 비트 -> 프레임 -> IP 패킷 -> TCP 세그먼트 -> HTTP 메시지
