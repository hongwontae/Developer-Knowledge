1. CPU (중앙처리장치)
=> 명령을 해석하고 실행하는 장치입니다.


2. CPU의 주요 역할
1. 연산 : 산술 연산, 논리 연산, 비교 연산..
2. 명령어 해석 : 프로그램이 실행하는 명령어를 해석하여 CPU가 이해할 수 있도록 변환합니다.
3. 제어 : CPU 내부의 모든 동 작을 제어합니다. 메모리, 입력 장치, 출력 장치
 간의 데이터 흐름을 조정합니다.


3. CPU 구성 요소
1. 산술 논리 연산 장치(ALU, Arithmetic Logic Unit)
=> 산술, 논리 연산을 수행하는 장치입니다.

2. 제어 장치 (CU, Control Unit)
=> CPU 내부 및 외부 장치를 제어하고 명령을 수행하는 장치입니다. 
=> 메모리에서 명령을 가져와 해석하고 실행하는 역할입니다.

3. 레지스터
=> CPU 내부의 고속 임시 저장소입니다.
=> 명령어, 연산 결과, 메모리 주소 등을 저장합니다.

3-1. 레지스터 종류
1. PC(Program Counter) : 다음에 실행할 명령어의 주소 저장합니다.
2. IR(Instruction Register) : 현재 실행 중인 명령어를 저장합니다.
3. ACC(Accumulator) : 연산 결과를 저장합니다.

4. 캐시 메모리
=> 자주 사용하는 데이터를 저장하는 초고속 메모리입니다.
=> CPU가 RAM보다 빠르게 접근할 수 있도록 도와줍니다.


4. CPU 동작 과정 (명령어 실행 사이클)
=> CPU는 명령어 사이클을 반복하여 동작합니다.

4-1. 인출 
=> 메모리(RAM)에서 명령어를 가져옵니다.
=> 프로그램 카운터가 명령어 주소를 가르킵니다.

4-2. 해석
=> 명령어를 해석하여 어떤 연산을 수행할지 결정합니다.

4-3. 실행
=> ALU에서 연산을 수행합니다.
=> 메모리에서 데이터 읽기/쓰기를 수행합니다.

4-4. 저장
=> 연산 결과를 레지스터 OR 메모리에 저장합니다.


5. CPU 성능을 결정하는 요소

5-1. 클럭 속도
=> 1초에 CPU가 처리할 수 있는 연산 횟수(단위 : GHz)

5-2. 코어 개수
=> CPU 안에 있는 독립적인 연산 장치 개수

5-3. 스레드 & 하이퍼스레딩
스레드 : CPU가 동시에 처리할 수 있는 작업 단위입니다.
하이퍼스레딩 : 1개의 물리 코어를 2개의 논리 코어처럼 활용합니다.

5-4. 캐시 메모리
=> L1 < L2 < L3.. 캐시 크기가 클수록 속도가 향상됩니다.

5-5. 버스 속도
=> CPU와 메모리, 기타 장치 간의 데이터 전송 속도를 의미합니다.
