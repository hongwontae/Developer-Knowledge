1. Protocol
=> 컴퓨터나 네트워크 장치들이 데이터를 주고 받을 떄 따르는 규칙이나 규약을 의미합니다.
=> 통신을 하기 위한 "약속"
=> 서로 데이터를 주고받을 때 같은 규칙을 따라야 원활한 통신이 가능합니다.


2. 프로토콜의 주요 역할
1. 데이터 형식 정의 → 어떤 방식으로 데이터를 주고받을지 정합니다.
2. 에러 감지 및 수정 → 데이터 전송 중 오류가 발생했을 때 대처 방법 제공합니다.
3. 데이터 전송 속도 조절 → 네트워크 속도에 맞춰 데이터를 조절해 손실 방지합니다.
4. 연결 및 종료 방식 규정 → 통신을 언제 시작하고 언제 끝낼지 결정합니다.


3. 네트워크 및 인터넷 관련 프로토콜
=> http, https, FTP, SMTP, POP3/IMAP, TCP/IP, UDP 등이 있습니다.


4. 프로토콜과 URL의 관계
=> 프로토콜://도메인 => 해당 부분이 프로토콜입니다.
=> https://www.google.com, ftp://example.com/file.txt, ws://example.com/socket
