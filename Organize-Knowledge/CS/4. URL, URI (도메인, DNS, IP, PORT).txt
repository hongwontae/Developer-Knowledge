1. URI(통합 자원 식별자)
=> 자원의 위치, 이름을 식별하는 문자열입니다.
=> **웹에서는 특정 리소스를 식별하기 위한 포괄적인 개념입니다.**
=> URI는 URL, URN을 포함하는 상위 개념입니다.


2. URL(통합 자원 주소)
=> URI의 한 종류로 *리소스의 위치와 접근 방법*을 포함합니다.
=> 웹에서 특정 자원의 위치(어디 있는지)와 접근 방식(어떻게 접근하는지)을 제공합니다.

=> 프로토콜://도메인[:포트]/경로/파일이름?쿼리스트링#프래그먼트
=> 도메인 다음 : 포트이고, /나오면 경로입니다.

2-1. 도메인
=> 서버(웹사이트)의 이름 또는 IP 주소를 의미합니다.
=> 사람이 기억할 수 있게 IP주소가 아닌 도메인을 사용합니다.
=> 웹 브라우저가 도메인을 IP 주소로 변환하기 위해 DNS를 사용합니다.

** DNS 변환 과정
=> 브라우저는 최근 방문한 IP 주소를 캐시합니다. 
=> 브라우저에 없다면 OS도 최근 사용된 DNS 정보를 캐시하기에 OS로 요청을 보냅니다.
=> 없다면 DNS 서버에 요청을 보냅니다.
=> DNS 서버가 도메인을 조회하여 IP 주소를 찾아 반환합니다.

** IP 주소
=> 인터넷에 연결된 장치(컴퓨터, 스마트폰, 서버 등) 또는 네트워크를 식별하는 고유한 숫자 주소입니다.
=> 네트워크 상의 위치를 나타내는 주소입니다.

1. IP 주소 종류
1-1. IPv4
=> 가장 널리 사용되는 IP 주소 형식입니다.
=> 32비트 주소 체계로, 4개의 10진수로 구분된 숫자 형태로 표시됩니다.

1-2. IPv6
=> 128비트 주소 체계로, 더 많은 주소를 제공합니다.
=> IPv4 주소 고갈 문제를 해결하기 위해 개발되었습니다.

2. IP 주소의 역할
=> 장치 식별
=> 데이터 전달 (데이터를 주고 받을 떄 보내는 장치와 받는 장치를 구분하는 역할을 합니다.)
=> 통신 경로 설정

3. IP 주소 유형
=> 공유된 공용 IP와 특정 네트워크에서만 유효한 사설 IP가 존재합니다.

3-1. 공용 IP
=> 인터넷에 연결된 모든 장치는 공용 IP를 가집니다.
=> 인터넷 상에서 유일한 주소여서, 다른 네트워크에서도 접근할 수 있습니다.

3-2. 사설 IP
=> 로컬 네트워크에서만 유효한 IP 주소입니다.
=> 주로 가정이나 회사의 내부 네트워크에서 사용됩니다.

** 사설 IP가 외부와 연결되는 방법 **
1. 용어 정리
1-1. 공유기 
=> 사설 네트워크와 인터넷을 연결하는 장치입니다.
=> 인터넷 서비스 제공업체로부터 공용 IP를 할당받습니다.
=> 해당 IP는 외부 인터넷과 연결되는 주소입니다.

1-2. NAT(Network Address Translation)
=> 사설 IP를 공유 IP로 변환합니다.
=> 사설 IP는 많을 수 있기 때문에(컴퓨터, 노트북, 핸드폰..) 이 요청들을 구분하기 위해 포트로 구분합니다.

2. 과정
사설 IP의 요청(컴퓨터) => 라우터는 공용 IP를 포함하고 있고 NAT를 실행 => 공용 IP 요청으로 변경되어 요청
=> 이 떄 사설 IP가 여러 개일 수 있으니 PORT 번호로 구별 => 응답 받고 사설 IP로 반환


2-2. Port
=> port 번호를 통해 서버(컴퓨터)가 어떤 서비스를 제공하는지 식별할 수 있습니다.
=> Node.js, Spirng server가 존재할 떄 http://localhost:8000 or http://localhost:6000으로 다른 서비스를
식별할 수 있습니다.

2-3. 경로
=> 서버 내부에서 특정 리소스(파일, 페이지, API..)의 위치를 나타냅니다.

2-4. detail
=> 컴퓨터나 서버는 ip 주소로 식별됩니다.
=> 같은 ip 내에서도 다양한 프로그램(서비스)들이 돌아가고 있습니다.
=> 그래서 포트 번호를 통해 각 서비스를 구분할 수 있습니다.


3. URN
=> 위치 정보 없이 리소스를 식별하는 개념입니다.
