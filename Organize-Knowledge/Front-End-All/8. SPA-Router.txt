1. SPA는 기본적으로 앱을 실행하는 JS 번들 파일을 다운로드합니다.
=> 모든 컴포넌트가 들어있거나 일부만 포함될 수 있습니다.
=> 코드 스플리팅에 따라 일부 컴포넌트는 필요할 떄 가져올 수 있습니다.
=> 기본 설정은 JS 번들 전체를 가지고 오는 것입니다.

2. SPA는 전환이 있을 떄 전체 새로고침없이 이미 가지고 있는 번들을 통해 컴포넌트를 렌더링합니다.
=> JS 번들이 컴포넌트가 되고 DOM을 업데이트 하는 방식입니다.
=> Virtual Dom을 사용하면 성능 최적화 + "화면에 깜빡이는 효과가 사라집니다."

3. 로딩 속도을 빠르게 하기 위해서 코드 스플리팅으로 초기 가지고 오는 번들의 크기를 줄이는 것이 중요합니다.
=> 코드 스플리팅으로 비어 있는 상태가 존재할 수 있기에 Suspense를 사용하여 로딩 상태를 표현합니다.


4. Router는 URL 기능을 사용하기 위해서 쓰여집니다.
=> 이 또한 코드 스플리팅이 없다면 전체 JS 번들을 가지고 옵니다.

5. JS Router URL
=> 서버에 요청하지 않고 클라이언트 내에서 URL을 동적으로 처리합니다.
=> 즉, JS Router가 URL을 처리하는 것입니다./ 동일한 index.html입니다.
