1. 데이터 패킷
=> 네트워크를 통해 전송되는 데이터의 기본 단위을 의미한다.
=> 네트워크 통신에서 데이터는 작고 관리 가능한 조각으로 분할되어 전송된다. => 이를 패킷이라고 한다.


2. 패킷의 구성 요소

2-1. 헤더
=> 패킷의 앞 부분으로 패킷의 목적지와 출발지, 순서, 오류 검사 정보 등 제어 정보를 포함한다.
=> 헤더의 내용은 프로토콜에 따라 다르다.
출발지 IP 주소, 목적지 IP 주소, 출발지 포트 번호, 목적지 포트 번호, 순서 번호(패킷의 순서, 재조립시에 사용된다.), 
제어 정보

2-2. 페이로드 
=> 패킷의 실제 데이터 부분

2-3. 트레일러
=> 일부 패킷은 트레일러라고 부르는 추가 정보를 포함할 수 있다.
=> 주로 오류 검출 및 수정에 사용된다.


3. 패킷 전송 과정
3-1. 분할 : 애플리케이션에서 생성된 큰 데이터를 네트워크를 통해 전달하기 위해 작은 패킷으로 분할한다.
3-2. 주소 지정 : 각 패킷에 출발지와 목적지 IP 주소 및 포트 번호를 지정한다.
3-3. 전송
3-4. 라우팅 : 네트워크 장치(라우터)는 패킷의 목적지 주소를 라우팅 테이블을 생성한다. 이를 토대로 경로를 다
음 경로를 결정한다.
3-5. 재조립 : 목적지에 도착한 패킷은 원래의 데이터 형태로 재조립된다. 패킷의 순서 번호를 사용하여 올바른 순서로 
조립한다.


4. 라우팅 테이블
4-1. 목적지 IP만으로는 그 위치에 효율적이고 정확하게 이동할 수 없다. 인터넷은 많은 네트워크와 장치로 
이루어져 있어 패킷이
어떤 **경로**를 통해 목적지로 도달할지 결정하는 매커니즘이 필요하다. => 이 떄 라우팅 테이블이 사용된다.
4-2. 라우팅 테이블은 라우터가 패킷을 어디로 보내야 할지를 결정하는 데 필요한 정보를 담고 있는 데이터베이스이다.
4-3. 네트워크 경로, 그 경로를 통해 도달할 수 있는 네트워크 장치(다음 홉, next hop)에 대한 정보

4-4. 라우팅 테이블 구조
1. 목적지 네트워크 : 도달해야 할 최종 네트워크 IP 주소 범위
2. 서브넷 마스크 : 네트워크와 호스트를 구분하기 위한 마스크
3. 게이트웨이/다음 홉: 패킷을 다음으로 전달할 라우터의 IP 주소
(다음 홉 => 패킷이 목적지로 가기 위해 특정 라우터를 거치는 것을 의미한다.)
4. 인터페이스: 패킷을 보내기 위해 사용할 라우터의 네트워크 인터페이스
