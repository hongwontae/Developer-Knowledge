1. 버블링
=> 이벤트가 발생한 요소에서부터 시작하여 부모 요소를 차례로 거쳐 최상위 부모 요소까지 전파되는 것을 의미합니다.
(만약 자식과 부모의 이벤트 타입이 다르면 무시한다.)
(만약 타입이 다르더라도 상위 요소의 이벤트가 존재하면 버블링하러 올라간다.)
(이벤트 리스너가 붙지 않더라도 해당 요소에 이벤트가 트리거되며 올라간다.)
(위의 3개의 개념은 캡쳐링도 동일하다.)


2. 캡쳐링
=> 이벤트가 최상위 부모 요소에서 시작하여 이벤트가 발생한 가장 하위 요소로 내려가는 과정을 의미합니다. 
=> 이벤트 리스너 세 번쨰 인자에 true를 설정하면 캡쳐링이 동작합니다.


면접 질문
버블링과 캡쳐링에 대해서 설명해보세요
=> 버블링은 하위 요소의 이벤트가 발생하면 최상위 부모 요소까지 전파하는 것을 의미합니다.
(event.stopPropagation() 호출하면 된다.)
=> 캡쳐링은 상위 요소의 이벤트가 발생하면 최하위 자식 요소까지 전파하는 것을 의미합니다.
(addEventListener의 세 번쨰 인자 true를 전달하면 캡쳐링이 작동합니다.)
