1. 동기
=> 동기란 작업이 순차적으로 실행되는 것을 입니다.
=> 하나의 작업이 완료되기 전에 다음 작업이 실행되지 않습니다.
=> 예를 들어, 가장 마지막에 실행된 함수가 완료되기 전까지 다음 함수가 실행되지 않는 흐름입니다.
=> 시간이 오래 걸리는 작업의 경우 프로그램이 멈춘 것처럼 느낄 수 있습니다.


2. 비동기
=> 작업이 즉시 실행되지 않고, 백그라운드에서 처리됩니다. 비동기 작업이 완료되면 콜백을 통해 결과를 처리합니다.
=> 이 떄 다른 작업들은 대기하지 않고 계속 실행됩니다.


면접 질문
동기와 비동기에 대해서 설명하세요
=> 동기 방식은 JS의 코드 전개 방식입니다. 작업이 순차적으로 이루어지는 것을 의미합니다.
=> 마지막에 호출된 함수의 작업이 완료되기 전까지 나머지 작업들은 이루어지지 않습니다.
=> 시간이 오래 걸리는 작업의 경우 프로그램이 멈춘것 처럼 보입니다.

=> 작업이 즉시 실행되지 않고, 백그라운드에서 처리됩니다. 이후 비동기 작업이 완료되면 콜백이나 프로미스, async/await
를 통해 결과를 처리합니다. 이 떄 다른 작업들은 대기하지 않고 계속 실행됩니다.
