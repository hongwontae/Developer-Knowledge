1. 불변성

2. 원시 데이터 불변성
=> 원시적 데이터 타입은 기본적으로 불변합니다.
=> 이 말은 메모리 상의 저장된 주소값의 값을 변경할 수 없다는 이야기입니다.
=> 그렇기에 새로운 공간을 만들어서 값을 할당함으로 값을 변경할 수 있습니다.


3. 참조 데이터 불변성
=> 참조 데이터의 경우에는 불변하지 않습니다.
=> 주소값을 그대로 할당하기 떄문입니다.


4. 불변성 유지
4-1. 얕은 복사와 깊은 복사할 떄를 구분해서 사용하면 불변성을 유지할 수 있습니다.
4-2. Immutable 라이브러리를 사용합니다.


면접 질문
불변성을 유지하려면 어떻게 해야할까요?
=> 객체나 배열을 복사할 떄 중첩된 객체, 배열 여부를 파악해서 깊은 복사할지 얕은 복사할지 구분하여 복사합니다.
=> Immutable, immer 라이브러리를 사용합니다.
