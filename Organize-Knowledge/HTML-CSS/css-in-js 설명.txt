1. CSS-in-JS
=> JS 파일내에서 CSS를 정의하고 적용하는 접근 방식입니다.
=> 주로 React 같은 컴포넌트 기반 라이브러리에서 사용됩니다.


2. 장점
2-1. 컴포넌트 기반 스타일링 : 한 파일에 컴포넌트와 스타일을 정의할 수 있다.
2-2. 스코프 안정성 : CSS-in-JS는 클래스 이름을 자동으로 유니크하게 만들어준다.
2-3. 조건부 스타일링 : JS의 삼항 연산자등을 통해 동적으로 스타일링이 가능합니다.
2-4. CSS 최적화 : 사용하지 않는 CSS 제거하는 트리 쉐어킹 => 번들 줄며 성능 향상
2-5. 스타일 재사용 : 스타일을 함수형으로 작성하여 재사용 가능

3. 단점
3-1. 런닝 커브 : 생소할 수 있다.
3-2. 성능 하락 : JS(컴포넌트 파일)에 style이 적용되어서 런타임 시간 소요, 그러기 떄문에 초기 페이지 렌더링이 늦어진다.
3-3. 디버깅 어려움
3-4. 개발 일관성
