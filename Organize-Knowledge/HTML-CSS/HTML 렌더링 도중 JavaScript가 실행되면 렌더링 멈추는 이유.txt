1. JavaScript가 기본적으로 동기적으로 실행되기 떄문입니다.
=> HTML이 브라우저에게 전달되어 먼저 파싱되더라도 script 태그를 만나서 JS 코드를 로드하게되면 이를 먼저 
실행한 뒤 html 파싱을 재개합니다


2. 이를 해결하기 위해 script 태그의 defer를 사용하는 것이 좋습니다.
=> defer를 사용하면 html 파싱도중 script 태그를 만나면 로드는 백그라운드에서 처리하게하고 html 파싱을 
멈추지 않습니다. JS 파일이 로드가 완료되었더라도 HTML 파싱이 끝나지 않았다면 기다린 후에 파싱 완료후 JS 파일이
실행됩니다.


면접 질문
html 렌더링 도중 JS 파일을 만나면 렌더링이 멈추는 이유
=> JS 기본 실행 방식이 동기적이기 떄문입니다. 브라우저에 html 파일이 전달되면 html 파싱을 시작하고 스크립트 태그를
만나서 이를 로드합니다. 이 떄 파싱이 중단되고 로드한 이후 실행까지 완료된 이후에 html 파싱을 다시 시작합니다.
=> 이러한 동작 방식의 문제점을 해결하기 위해 스크립트 태그의 defer를 사용하면 됩니다.
