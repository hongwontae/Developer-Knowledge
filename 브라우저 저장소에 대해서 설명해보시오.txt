1. 로컬 스토리지, 세션 스토리지, 쿠키가 존재한다.

브라우저 저장소는 두 가지이다.
웹 스토리지(Web Storage) : 웹 데이터를 클라이언트에 저장하기 위해 만들어진 저장소./ 최대 5MB까지 지원
쿠키(Cookie) : 서버와 클라이언트 간의 지속적인 데이터 교환을 위해 저장소/ 하나의 사이트에서 저장할 수 있는 쿠키 크기는 4KB, 최대 쿠키수는 20개이다.

2. 로컬 스토리지와 세션 스토리지는 HTML5에 추가된 저장소로 간단한 key와 value를 저장할 수 있고 두 개의 차이점은 영구성에 있다. 

3. 로컬 스토리지는 사용자가 지우지 않는 한 브라우저에 남아있지만 세션 스토리지는 윈도우나 창을 닫으면 삭제된다.

4. 로컬은 도메인만 같으면 브라우저가 달라도 전역적으로 데이터가 공유되서 저장, 조회, 변경 등이 가능해.
   세션은 메인이 같더라도 브라우저가 다르면 (탭 브라우저, 다른 브라우저) 각각의 세션 스토리지가 형성되서 데이터 공유가 되지 않아서 저장, 조회, 변경 등이 불가능해.

4. 쿠키란 만료기한이 있는 key-value 저장소이다.

5. 로컬 스토리지와 세션 스토리지의 key-value에 저장되는 값은 모두 문자열이다. JSON.stringify를 사용해서 객체를 한 번에 저장할 수 있고 JSON.parse를 이용해 받을 수 있다. => 객체를 저장할 수 있는 것이 장점
(쿠키는 단순 문자열만 가능)

6. 장점은 서버에 불필요한 데이터를 저장하지 않는다./ 단점은 HTML5를 지원하지 않는 브라우저의 경우 사용이 불가

7. 쿠키는 서버와 지속적으로 데이터 교환을 하기 위해 만들어졌다. 쿠키는 4KB의 용량 제한이다. 한 사이트당 20개의 쿠키를 가질 수 있다. 만료기간 지정 가능/ 쓸데없는 값이 증가하면 네트워크 트래픽이 증가한다. 만료기간 존재하여 사라진다. 영구 쿠기 => 만료 기간 멀게 하여 설정

8. 차이점
웹 사이트에서 쿠키를 설정하면 이후 모든 웹 요청은 쿠키정보를 포함하여 서버로 전송된다.
(request 할 떄 request Header에 자동으로 넣어 서버에 데이터를 전송한다)
Web Storage는 저장된 데이터가 클라이언트에 존재할 뿐 서버로 전송은 이루어지지 않는다. 이는 네트워크 트래픽 비용을 줄여 준다.
