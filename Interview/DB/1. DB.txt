1. DB에서 정규화란 무엇일까요?
=> DB 설계에서 데이터 중복을 줄이고 무결성을 유지하기 위해 데이터를 구조화하는 과정입니다.


2. 정규화의 목적
2-1. 데이터 중복 최소화할 수 있습니다.
2-2. 이상을 제거할 수 있습니다. 
2-3. 데이터 무결성을 유지할 수 있습니다.
2-4. 데이터 독립성을 향상할 수 있습니다.


3. 정규화의 과정
=> 제 1 정규형 => 제 2 정규형 => 제 3 정규형 => BCNF => 4NF => 5NF 순서로 진행됩니다.

3-1. 제 1 정규형
=> 테이블의 모든 칼럼 값이 원자값만 포함해야 합니다.
=> 각 속성이 더 이상 쪼개질 수 없는 최소 단위로 저장되어야 합니다.
=> EX) 전화번호 칼럼에 전화번호가 2개 들어가 있는 상황을 의미합니다.

3-2. 제 2 정규형
=> 1NF을 만족하고, 기본 키의 부분집합이 결정자가 되는 *부분적 함수 종속*을 제거하는 것을 의미합니다.
=> 기본 키의 일부에만 종속된 컬럼이 없어야 합니다. (모든 칼럼이 기본 키에 종속되어야 합니다.)

3-3. 제 3 정규형
=> 2NF을 만족하고, 비 기본 속성이 기본 키가 아닌 다른 속성에 종속되지 않아야 합니다.
=> *이행적 종속*을 제거해야 합니다.
(X=>Y=>Z 종속 형태가 있을 떄 Z는 X가 아닌 Y에 종속된 것이므로 문제 발생합니다. 이 떄 Y, Z를 별도의 table로 분리
해야합니다.) (기본키가 아닌 칼럼이 다른 일반 칼럼에 종속되지 않도록 테이블을 분리합니다.)

3-4. BCNF(보이스 코드 정규형)
=> 3NF을 만족하고, 모든 결정자가 후보 키여야 합니다.
=> 후보 키가 아닌 결정자가 존재하면 안됩니다.

3-5. 4NF, 5NF
=> 4NF : 다치 종속을 제거합니다.
=> 5NF : 조인 종속을 제거하여 테이블을 더 세분화합니다.


4. 정규화의 특징
=> 데이터 무결성이 보장되지만, 성능 저하 가능성이 존재합니다.
=> 실제 DB 설계는 정규화와 비정규화를 적절히 조합하여 성능을 고려합니다.
