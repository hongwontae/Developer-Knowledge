1. 바닐라 JS 이벤트와 React 이벤트의 차이

1-1. 이벤트 제거 방식
JS => removeListener를 통해 명시적으로 제거해야 합니다.
React => 컴포넌트가 언마운트될 떄 자동으로 이벤트가 정리됩니다.

1-2. 이벤트 객체
JS => 브라우저 네이티브 이벤트 객체를 받습니다.
React => SyntheticEvent 객체를 받습니다. 브라우저 이벤트를 감싸는 래퍼 객체이고 이 객체를 재사용합니다.

1-3. 이벤트 리스너 등록 방식
JS => addEventListener를 통해 등록합니다.
React => JSX의 이벤트 속성에 바인딩하여 사용합니다.

1-4. 버블링, 캡쳐링
JS, React 모두 버블링과 캡쳐링이 존재합니다.

1-5. this
JS => 바닐라 JS에서 이벤트 핸들러 내부의 this는 이벤트를 받은 요소를 가르킵니다.
React => 함수형 컴포넌트에서는 this를 사용하지 않고 useState, useEffect 훅을 사용하여 상태를 관리하기
때문에 this를 사용할 일이 없습니다.

1-6. 이벤트 위임의 차이점
JS => 직접 구현
React => 모든 이벤트가 document에서 캡쳐링, 버블링 처리됩니다./
