1. ⭐ 리플로우와 리페인트에 대해 설명해주세요.
=> 리플로우는 요소의 크기, 위치, 레이아웃이 변경될 때 발생하며, 비용이 가장 큰 연산입니다.
=> 리페인트는 레이아웃은 그대로지만 색상, 배경, 그림자 같은 스타일만 변경될 때 발생합니다.


4. Webpack, Babel, Polyfill에 대해 설명해주세요.
=> Webpack은은 모듈 번들러로, JS, CSS, 이미지 등을 하나의 번들 파일로 묶어 최적화합니다.
=> Babel은 최신 JavaScript 코드를 구형 브라우저에서도 실행할 수 있도록 변환하는 트랜스파일러입니다.
=> Polyfill은 브라우저에서 지원하지 않는 기능을 추가합니다. (예: fetch, Promise 등)


5. ESLint와 Prettier에 대해 설명해주세요.
=> ESLint는 코드 품질을 유지하기 위한 JavaScript 린터, 문법 오류나 일관성 문제를 자동으로 감지합니다.
=> Prettier는 코드 스타일을 강제하는 코드 포매터, 자동으로 들여쓰기와 공백을 조정합니다.


6. SPA와 MPA에 대해 설명해주세요.
=> SPA는 한 개의 HTML 파일로 동작하고, 페이지 이동 없이 비동기 데이터 로딩을 수행합니다.
=> MPA는 각 페이지마다 새로운 HTML을 요청하는 전통적인 방식입니다.


7. bundle의 사이즈를 줄이려면 어떻게 해야 하나요?
7-1. 코드 스플리팅을 사용합니다. (React.lazy, import)로 필요할 떄 로드합니다.
7-2. 이미지 최적화
7-3. 라이브러리 최적화합니다. 무거운 라이브러리는 일정 부분만 import 합니다.
7-4. 캐싱 활용 → CDN 사용, 정적 리소스 캐싱
