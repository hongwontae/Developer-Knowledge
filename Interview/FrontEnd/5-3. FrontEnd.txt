3. 객체 지향 프로그래밍이란 무엇인가요?
=> 객체를 중심으로 데이터를 구조화하고 재사용성을 높이는 프로그래밍 패러다임입니다.

4. 객체 지향 프로그래밍의 특징에 대해 말씀해주세요.
4-1. 캡슐화(Encapsulation): 객체 내부 상태를 보호하고, 외부에서 직접 접근을 제한합니다. (#, TS에서 private )
4-2. 상속(Inheritance): 기존 클래스의 기능을 재사용하여 새로운 클래스를 생성합니다. (extends, super)
4-3. 다형성(Polymorphism): 같은 메서드명이 다양한 방식으로 동작 합니다. (오버라이딩)
4-4. 추상화(Abstraction): 불필요한 세부 정보를 숨기고 구체적인 기능은 자식 클래스가 구현합니다.
(throw new Error()로 강제할 수 있습니다.)

(EIPA)

5. 객체 지향 프로그래밍의 장단점에 대해 말씀해주세요.
=> 장점 : 코드 재사용성 증가, 유지보수가 용이, 현실 세계 모델링에 적합합니다.
=> 단점 : 설계 복잡도가 높아지거나 절차지향보다 속도가 느릴 수 있습니다.

7. BOM과 DOM에 대해 설명해주세요.
=> BOM(브라우저 오브젝트 모델)은 브라우저 창과 관련된 객체입니다.
=> DOM은 HTML 문서를 JS에서 조작할 수 있도록 트리 형태로 변환한 구조입니다.
=> DOM은 HTML 요소를 조작하고 BOM은 브라우저 환경을 조작합니다.


8. 리플로우와 리페인트의 발생 조건에 대해 설명해주세요.
=> 리플로우는 레이아웃 변경할 떄 발생 합니다. (width, height, position)
=> 리페인트는 시각적 속성이 변경할 떄 발생합니다. (background-color, color)


9. visibility 속성은 리플로우를 발생시킬까요?
=> 리페인트만 발생시키고 리플로우는 발생하지 않습니다. (요소의 위치나 크기가 바뀌지 않습니다.)
=> 다만 display : none의 경우 리플로우가 발생합니다.
