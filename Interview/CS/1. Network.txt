3. ⭐ HTTP의 GET, POST를 비교 설명해주세요.
=> get은 데이터 조회, post는 데이터 생성/전송
=> get은 데이터를 전달할 떄 URL 쿼리 파라미터에 포함하고 post는 body에 포함
=> get은 URL이 노출되어 보안에 취약하고 POST는 상대적으로 안전


4. HTTP의 PUT, PATCH를 비교 설명해주세요.
=> PUT은 리소스 전체 수정, patch는 리소스를 부분 수정
=> 사용자의 모든 정보 업데이트 vs 사용자의 전화번호만 업데이트


5. ⭐ HTTP의 Status Code에 대해 설명해주세요.
=> 서버가 클라이언트 요청에 대한 응답 정보를 숫자로 표현한 것입니다.
=> 1xx : 요청 중
=> 2xx : 성공
=> 3xx : 리다이렉트
=> 4xx : 클라이언트 오류
=> 5xx : 서버 오류


6. ⭐ HTTP와 HTTPS의 차이점에 대해 설명해주세요.
=> HTTP는 암호화되지 않고 HTTPS는 SSL, TLS에 의해 암호화가 적용됩니다.
=> HTTP 기본 포트는 80, HTTPS는 443입니다.
=> HTTPS는 민감한 데이터를 보호할 떄 주로 사용하는 프로토콜입니다.


7. HTTPS의 동작 방식에 대해 설명해주세요.
=> 클라이언트가 서버에 HTTPS 요청
=> 서버가 TLS, SSL 인증서를 제공
=> 클라이언트가 인증서를 검증합니다.
=> 세션 키를 생성하고 서버와 공유합니다.
=> 이후 데이터는 대칭 암호화 방식으로 통신합니다.


8. ⭐ 프로세스와 스레드를 비교 설명해주세요.
=> Process : 운영체제로부터 자원을 할당받은 작업의 단위입니다.
=> Thread : 프로세스가 할당받은 자원을 이용하는 실행 단위입니다.
(프로세스는 회사, 스레드는 직원)


9. Stack과 Queue의 차이점
=> Stack은 후입선출, Queue는 선입 선출 흐름을 가집니다.
