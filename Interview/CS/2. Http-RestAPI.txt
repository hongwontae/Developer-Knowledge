1. Http(HyperText Transfer Protocol)
=> 브라우저와 서버간에 데이터를 주고 받을 떄 사용하는 프로토콜 입니다. (프로토콜은 통신 규약입니다.)

2. http 특징
2-1. 비연결성
2-2. 무상태성
2-3. 확장성 (header를 통해 여러 metadata를 추가하거나 형식을 지정할 수 있습니다.)
2-4. 요청-응답 기반 구조입니다.
2-5. 캐싱 지원 (image, css는 한 번 받아두고, 다시 요청하지 않아도 보여줍니다.)

3. 클라이언트에게 전달되는 것은 응답 메시지입니다.
=> 상태줄, 헤더, 본문(HTML, Image, css, JS)
=> 즉, 크게 상태줄, 헤더, 본문을 전달합니다.


4. Rest API
=> HTTP을 잘 사용하기 위한 하나의 방식입니다. (웹에서 리소스를 다루기 위한 아키텍쳐)
=> HTTP 메서드와 URL 구조를 활용해서 서버의 자원을 클라이언트가 요청하고 조작할 수 있는 방식입니다. (http 메서드, url 구조)

4-1. Rest API의 6가지 원칙
1. 클라이언트-서버 구조: 역할 분리
2. 무상태성 (Stateless)
3. 캐시 처리
4. 계층화 처리
5. 일관된 인터페이스
6. 리소스 기반

=> http 메서드와 url 구조를 사용해서 정해진 규칙성을 부여하여 서버 - 클라이언트 통신에 활용합니다.
