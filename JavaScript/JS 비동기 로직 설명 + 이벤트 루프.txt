1. 자바스크립트에서 비동기 로직이 어떻게 동작하는지 설명해주세요. (이벤트 루프)
=> JS는 싱글 쓰레드 언어지만 비동기 로직을 통해서 효율적으로 작업을 처리할 수 있습니다.
=> 해당 과정은 이벤트 루프를 통해 관리됩니다.


2. 이벤트 루프와 비동기 로직
2-1. JS의 메인 쓰레드는 콜 스택을 통해 함수 호출을 관리합니다.
2-2. 함수가 호출되면 콜 스택에 추가되고 완료되면 제거되는 흐름입니다. 이 과정에서 가장 마지막에 호출된 함수가 먼저
제거되는 후입선출되는 흐름을 가집니다. 기본적으로 동기적으로 동작합니다.


3. WEB APIs, 비동기 처리 시스템
3-1. 비동기 작업 자체는 웹 API, 비동기 처리 시스템에 전달되어 실행됩니다.
3-2. 비동기 작업이 완료되면 태스크 큐 또는 마이크로태스크 큐로 전달됩니다.


4. 태스크 큐, 마이크로태스크 큐
4-1. 큐에 있는 비동기와 관련된 콜백은 콜 스택이 비워져 있다면 가져와서 실행됩니다.
4-2. 이 작업은 이벤트 루프에 의해서 관리되고 실행됩니다.


면접 질문
JS 비동기 로직 설명 + 이벤트 루프
=> JS는 싱글 쓰레드 언어라 동기적으로 동작합니다. 콜 스택에서 함수 호출를 관리하고 가장 먼저 호출된 함수가 가장 먼저
제거되는 후입선출되는 흐름을 갖습니다. 하지만 비동기 작업을 통해 이를 효율적으로 관리할 수 있습니다.
=> 비동기 함수가 호출되면 비동기 함수 자체는 WEB API나 비동기 처리 시스템에 전달되고 비동기 작업이 완료되면 해당
콜백은 태스크 큐나 마이크로 태스크 큐에 전달됩니다. 해당 콜백들은 콜 스택이 비워져 있을 떄 콜 스택에 전달되어서
실행됩니다. 이 작업은 이벤트 루프에 의해서 관리되고 실행됩니다.
=> 이런 방식을 통해 멀티 쓰레드처럼 사용할 수 있습니다.
