1. 태스크 큐와 마이크로태스크 큐에 대해서 설명하세요.
=> JS에서 비동기 작업을 관리하는 두 가지 주요 큐입니다.


2. 태스크 큐
=> 비동기 작업이 완료되면 해당 콜백 함수가 전달되는 큐입니다.
=> 해당 큐에 들어가는 함수들은 Browser API 비동기 작업, requestAnumationFrame의 콜백 함수입니다.


3. 마이크로 태스크 큐
=> 태스크 큐의 작업보다 먼저 처리되는 큐입니다.
=> 프로미스와 async/await에 관련된 작업들이 마이크로 태스크 큐에 들어갑니다.
=> Promise.then, catch이나 Promise.finally, MutaionObserver(DOM 변경을 관찰하고 관련된 콜백 함수를 의미합니다.)


면접질문
=> 태스크 큐와 마이크로태스크 큐에 대해서 설명하세요
=> 태스크 큐는 Brwoser API와 requestAnimationFrame의 비동기 작업과 관련된 콜백함수가 전달되는 공간입니다.
=> 비동기 작업이 완료된 이후 관련된 콜백이 태스크 큐에 도착하고 콜 스택이 비워져 있다면 콜 스택에 전달되고 
호출됩니다. 이는 이벤트 루프에 의해서 관리되고 실행됩니다.
=> 마이크로테스크 큐는 Promise, async/await에 관련된 then 콜백, awiat 함수 다음 코드가 전달되는 공간입니다.
=> 마이크로테스크 큐는 테스크 큐보다 작업 우선순위가 더 높습니다. 이 경우에도 이벤트 루프에 의해서 콜 스택이 비워져
있다면 전달되어 실행됩니다.
