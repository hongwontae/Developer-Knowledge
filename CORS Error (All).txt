1. Origin
=> 프로토콜 + Host + 포트번호의 튜플
=> 포트번호 생략 가능(HTTP : 80, HTTPS : 443)
=> 셋 중에 하나라도 다르면 다른 Origin이다.


2. 프로토콜
=> 통신 규칙


3. Host
인터넷에 접속되어 있는 각각의 컴퓨터를 가리키는 주소


4. 포트
IP주소는 컴퓨터를 찾을 떄 필요한 주소
포트는 컴퓨터 안에서 프로그램을 찾을 떄 필요한 주소
=> IP를 사용하는 프로그램이 여러 개라면 하나의 IP주소로는 분간하기 어렵다. 그래서 포트번호가 필요한 것이다. 


(5,6은 다른 출처로부터 리소스의 요청을 제한하는 정책이다.)

5. 동일 출저 정책(Same Origin Policy)
대부분의 웹 브라우저는 Same Origin Policy(SOP)라는 보안 정책을 준수한다.
=> 사이트 접속(고유 Origin 존재) => 다른 Origin이 접속한 사이트에 대해 요청 => 브라우저가 자동으로 거절


6. 교차 출처 리소스 공유(Cross-Origin Resource Sharing)
출처가 다르더라도 리소스 접근 허용 => CORS를 정책을 지킨 리소스 요청
=> SOP을 어기더라도 CORS를 지키면 일부 리소스 접근가능


7. 브라우저
SOP, CORS는 브라우저가 컨트롤한다. 
서버가 CORS를 무시한 접근이지만 응답을 한다. => 브라우저가 CORS 위반을 파악하고 요청자체를 파기시킨다.
브라우저를 거치지 않은 요청이고 서버의 CORS 로직이 없다면 다른 출저라도 응답해준다.
