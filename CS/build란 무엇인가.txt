1. Build
=> 소스 코드와 자원들을 최적화된 형태로 변환하여 실제 실행 가능한 형태로 준비하는 과정
=> 개발 중인 애플리케이션을 프로덕션 환경에서 실행할 수 있도록 준비하는 단계

1-1. 단계
1. 컴파일 : JS, TS 소스 코드들을 브라우저 친화적인 형태로 변환한다. (Babel을 통해 최신 문법 지원x 브라우저에서 코드 통하게)
2. 모듈 번들링 : 여러 개의 파일을 하나의 파일로 변환 (Webpack, Rollup, Parcel을 통해 변환)
3. 정적 자원 최적화 : 압축 및 최적화(js, css, html, image), 파일분할(필요할 떄 로드시키기 위함)
4. 페이지 렌더링 : SSR, SSG
5. 환경 설정 : 개발과 deployment에 따라 다른 환경 변수 제공
