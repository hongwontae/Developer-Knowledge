1. 상대경로
=>  현재 작업 디렉토리(혹은 기준점)를 기준으로 파일이나 디렉토리의 위치를 지정합니다. 
ex) ./file.txt : 현재 작업 디렉토리 내의 file.txt를 의미한다
    ../folder/file.txt : 부모 디렉토리의 file.txt
    folder/file.txt : 현재 디렉토리의 folder 폴더의 file.txt, 즉 자식 디렉토리의 파일을 의미한다.


2. 절대경로
=> 파일 시스템의 루트부터 시작하여 파일이나 디렉토리의 정확한 위치를 지정한다.
=> 절대경로는 경로의 시작점이 고정되어 있기 때문에, 어디서 접근하든 동일한 파일, 디렉토리를 지칭한다.
ex) C:\Users\User\Documents\file.txt => Windows 시스템의 c 드라이브내의 경로의 파일
    /home/user/documents/file.txt => UNIX 계열 


3. Unix vs Windows

3-1. Unix
1. 경로 구분자 : /(슬래시)
2. 경로 표기법 : 절대경로는 루트 디렉토리(시스템에 의해 정의된다.), 상대경로는 현재 작업 디렉토리부터 출발
3. 파일이나 폴더의 대소문자 구분한다.

3-2. Windows
1. 경로 구분자 \ (역 슬래시)
2. 절대경로, 상대경로 결정은 Unix와 동일하다.
3. 대소문자 구분하지 않는다.

3-3. 경로 구분자 해결
=> NodeJS에서 path 모듈을 통해서 운영 체제 상관없이 경로 처리 가능
