1. 프로토콜이란?
1-1. 프로토콜(protocol)은 컴퓨터 네트워크에서 데이터를 교환하기 위한 규칙과 절차의 집합입니다.
네트워크 프로토콜은 데이터를 어떻게 포맷하고, 송신하고, 수신하는지, 오류를 어떻게 처리하는지 등을 정의합니다. 

1-2. 특징
1. 규칙과 규격
2. 표준화
3. 계층적 구조
4. 오류 처리와 보안


2. 계층
2-1.  데이터 통신을 보다 효율적이고 체계적으로 관리하기 위해 여러 단계를 나누어 각각의 단계에서 
특정 기능을 수행하는 방식을 말합니다.

2-2. 이 계층 모델은 각 계층이 독립적으로 작동하면서도 상호 작용할 수 있도록 설계되어 있다.

2-3. OSI 모델
=> 네트워크 통신을 7개의 계층으로 나누어 설명한다. 각 계층은 특정 네트워크 기능을 담당하며, 
아래에서 위로 쌓아 올려진다.
1. 물리계층
2. 데이터 링크 계층 
3. 네트워크 계층 (Network Layer)
4. 전송 계층 (Transport Layer)
5. 세션 계층 (Session Layer)
6. 표현 계층 (Presentation Layer)
7. 응용 계층 (Application Layer)

2-4. TCP/IP 모델
=> 인터넷에서 주로 사용되는 프로토콜을 설명하기 위해 만들어졌으며,  4개의 계층으로 나누어진다.
1. 네트워크 인터페이스 계층 (Network Interface Layer)
2. 인터넷 계층 (Internet Layer)
3. 전송 계층 (Transport Layer)
4. 응용 계층 (Application Layer)


3. 계층 모델의 장점
3-1. 모듈화
3-2. 문제 해결 용이성
3-3. 상호 운용성
3-4. 관리 용이성
