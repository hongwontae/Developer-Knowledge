1. React 에서 상태 변화가 생겼을 때, 변화를 어떻게 알아채는지에 대해 설명해주세요.

1-1. 상태 변경 함수 호출
=> setState, this.Setstate를 호출하면 React는 상태 변경 요청을 감지하고, 컴포넌트를 업데이트해야 한다고 예약합니다.

1-2. 가상 DOM 업데이트
=> 상태 변경 이후, 새로운 가상 DOM를 생성합니다.

1-3. 변경 사항 비교
=> 새로운 가상 DOM과 이전 가상 DOM을 비교한 뒤 변경된 부분만 실제 DOM에 적용합니다.


면접 질문
React에서 상태 변화가 생겼을 떄, 변화를 어떻게 알아채는지에 대해 설명해주세요.
=> setState를 호출했을 떄 React는 상태 변경 요청을 업데이트 큐에 추가합니다. 해당 호출은 비동기적으로 처리됩니다.
(업데이트 큐는 여러 업데이트 요청을 모아서 한 번의 리렌더링 사이클에서 처리합니다.)
=> 상태 업데이트로 인해서 상태가 변경되면 React는 이전 가상 dom과 현 가상 dom을 비교해서 변경된 사안만 수정하여
실제 dom에 반영합니다.  

※ setState로 상태 변경을 예약하고 비동기적으로 호출되어 상태가 변화되면 React 내부 알고리즘에 의해서 컴포넌트가
리렌더링됩니다. => 이게 낫겠다.
