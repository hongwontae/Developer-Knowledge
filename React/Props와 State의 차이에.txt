1. props
=> 부모 컴포넌트가 자식 컴포넌트에게 내려주는 읽기 전용 데이터입니다.
=> 데이터 전달이나 목적으로 사용됩니다.


2. state
=> 컴포넌트 내부의 데이터입니다.
=> 데이터나 UI 상태 관리에 사용됩니다.
=> setState를 통해 상태를 업데이트 할 수 있습니다.


면접 질문
Props와 State의 차이에 대해 설명해주세요.
=> Props는 부모에서 자식을 전달되는 읽기 전용 데이터입니다. 부모 컴포넌트의 변경이 자동으로 반영됩니다.
=> state는 컴포넌트 내부의 데이터이고 상태를 변경할 수 있고 이를 통해 업데이트 할 수 있습니다.
=> 즉, props는 데이터 전달 목적이고 state는 동적 데이터와 UI 상태 관리 목적입니다.
