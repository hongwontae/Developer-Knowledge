1. virtaul dom은 실제 dom의 복사본으로 JS 객체로 표현되고 메모리에 저장되어 관리됩니다.
2. 상태 변화가 일어나면 이전의 virtaul dom과 기존 virtual dom의 비교가 발생하기 떄문에 실제 DOM을 비교하는 것보다 
빠르고 효율적으로 업데이트가 발생합니다.

TMI
=> 전체 가상 DOM 트리끼리의 비교이다.
=> 상태가 변경되면 그 상태 변경부분 가상 돔 트리 트리 + 전체 가상 돔 트리가 합쳐지고 이전 돔 트리와 비교되는 것이다.
=> JSX가 JS 객체로 변환되어 가상 DOM 트리 형성

가상 DOM 트리는 메모리에 저장?
=> 애플리케이션이 실행되는 브라우저 메모리를 의미한다.
=> 해당 공간은 JS 실행되는 공간


면접 질문-1
Virtual DOM 이 무엇인지 설명해주세요
=> 가상 DOM은 실제 DOM을 메모리에서 추상화한 객체입니다.
=> DOM의 사본으로 요소들을 JS 객체로 표현합니다.


면접 질문-2
Virtual DOM 작동 원리에 대해 설명해주세요
=> React 컴포넌트가 렌더링 될 떄 JSX가 JS 객체 형태로 Virtual DOM을 이루고 이를 기반으로 실제 DOM이 표현됩니다.
=> 상태의 변화가 일어나면 변경된 상태를 반영한 Virtual DOM이 만들어집니다.
=> 그리고 이전의 Virtual DOM과 변경 상태를 반영한 Virtaul DOM을 비교하고 변경된 상태만 실제 DOM에 적용합니다.
=> 이 같은 기능은 변경된 부분만 업데이트하여 렌더링 성능을 최적할 수 있습니다.
